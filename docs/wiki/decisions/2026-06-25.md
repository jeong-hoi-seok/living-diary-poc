# 2026-06-25

메인 화면에 Blender 3D 태양계(`unive.blend`)를 펜화(NPR) 스타일로 렌더링하고 궤도 애니메이션을 재생. 렌더러 선택에서 New Architecture 제약으로 한 차례 전면 교체가 있었음. ([[2026-06-24]] 초기 셋팅 위에 작업)

## 결정 | 3D 에셋은 `.blend`가 아니라 `.glb`로 변환해 사용
- RN은 `.blend`를 직접 못 읽음 → Blender headless로 glTF(GLB) 익스포트.
- `metro.config.js`의 `resolver.assetExts`에 `glb`/`gltf` 추가(기본 미포함).
- 애니메이션: 초기엔 액션별 분리 트랙(행성 5개). 재생 시 전 트랙을 동시에 돌려야 5개가 함께 공전.

## 결정 | 렌더러: react-native-filament → **three.js WebGPURenderer + react-native-wgpu**
- 1차 시도: `expo-gl` + `@react-three/fiber/native` + drei.
  - **실패**: New Architecture(Fabric)에서 expo-gl `GLView`(레거시 뷰)가 interop 합성에 실패 → gl 컨텍스트·프레임 루프는 정상인데 화면이 완전 검정. 우리 코드 문제 아님.
  - Reanimated 4가 New Architecture 필수라 new arch를 끌 수도 없음.
- 2차 시도: `react-native-filament`(Fabric 네이티브, GLB+애니 전용).
  - 렌더는 됨. 그러나 원본이 **Blender EEVEE 전용 NPR**(툰 Shader-to-RGB, 잉크 외곽선, emission, Standard 톤맵)이라 filament(PBR 전용, **ACES 톤맵 고정, JS로 톤맵/노출 제어 불가**)로는 룩 재현 불가. 종이가 흰색이 안 되고 명암이 ACES에 뭉개짐.
- **최종 채택: three.js `WebGPURenderer`(`three/webgpu`) + `react-native-wgpu`**.
  - new arch에서 동작하는 Fabric 네이티브 GL 서피스(WebGPU/Metal). expo-gl 검정 문제 회피.
  - **Apple Silicon Mac의 iOS 시뮬레이터에서도 렌더됨**(Metal).
  - `renderer.toneMapping = NoToneMapping` → Blender "Standard"처럼 밝은 값이 흰색으로 클립.
  - 설치: `react-native-wgpu` + `react-native-webgpu`(셰임이 podspec을 nested로 숨겨 top-level 직접 설치로 autolink 보장), `three@0.184`.
  - `react-native-webgpu`의 optional peer(worklets) 충돌 회피 위해 `npm install --legacy-peer-deps` 필요.

## 결정 | NPR(펜화) 룩은 머티리얼 교체로 재현 — GLB 재익스포트 불필요
- glTF엔 PBR로 베이크돼 들어오므로, 로드 후 **머티리얼 이름**(`Toon_Planet`/`Ink_Sun`/`Orbit_Line`)으로 매칭해 교체.
  - 노드/메시 이름은 한글("구체.007")이라 불안정 → 머티리얼 이름 매칭이 견고.
- 행성: `MeshToonMaterial` + 2단계 `gradientMap`(NearestFilter). Blender ColorRamp(CONSTANT 2-stop) 값을 그대로 옮김 — **그림자 0.6 회색 / 빛 1.0 흰색, base 흰색**. (lit 면이 순백이라 흰 배경에 녹고, 그림자+외곽선으로 형태가 보이는 게 정상.)
- 외곽선(잉크): **inverted-hull 코드 생성** — 지오메트리를 공유하는 `MeshBasicMaterial`(검정, `BackSide`) 메시를 원본의 자식으로 붙이고 로컬 스케일만 키움(`OUTLINE_HULL_SCALE`).
  - unlit이라 라이트와 무관하게 균일 검정 → 그라데이션 없는 또렷한 펜선.
  - postprocessing(OutlinePass)·EdgesGeometry는 배제(불안정/구체에 부적합).
- 배경: 흰색(`scene.background`/clearColor). 궤도선: unlit 검정.
- 카메라: Blender 씬 카메라를 Y-up 변환한 값(`[0,15,19]`, 50mm)으로 초기화.

## 결정 | Blender MCP로 원본 머티리얼 값 직접 추출
- 작업 중 Blender가 켜져 있어 MCP(`execute_blender_code`)로 `Toon_Planet` 노드 그래프·ColorRamp·라이트·월드를 직접 읽어 앱 값과 대조 → 추측 없이 1:1로 맞춤.

## 결정 | 카메라 조작은 gesture-handler 직접 제어
- drei/three OrbitControls는 RN(DOM 이벤트 의존)에서 불안정 → `react-native-gesture-handler`로 직접.
- Pan=궤도 회전, Pinch=줌, 우하단 +/- 버튼=줌. 제스처 콜백은 `.runOnJS(true)`로 JS 스레드에서 ref만 갱신 → `setAnimationLoop`에서 카메라 반영(three 객체를 UI worklet 런타임과 섞지 않음).
- 루트를 `GestureHandlerRootView`로 래핑(`_layout.tsx`).

## 결정 | 메인 화면 풀스크린 + 하단 타이틀 오버레이
- `SceneViewer`가 화면 전체. 타이틀/설명은 하단 절대배치 오버레이(`pointerEvents="none"`로 제스처/버튼 통과).

## 결정 | dev 스크립트: build 그룹 / dev 그룹 분리
- `expo prebuild`가 `ios`/`android` 스크립트를 `expo run:*`(네이티브 빌드, **QR 없음**)로 덮어써 QR이 사라졌었음.
- 이름으로 두 그룹 명확히 구분:
  - `build:ios` / `build:ios:device` / `build:android` = `expo run:*`(네이티브 빌드·설치, 가끔).
  - `dev:ios` / `dev:android` / `dev` = `expo start --dev-client ...`(개발 서버·QR, 평소).
- **`--dev-client` 플래그 필수**: `expo-dev-client` 패키지가 없으면 `expo start`가 Expo Go 모드 기본 →
  네이티브 모듈 없는 Expo Go에서 열려 `react-native-webgpu` import 크래시. 플래그로 dev client 타겟 강제(기존 dev 빌드로 동작, 재빌드 불필요).
- **이 앱은 wgpu 네이티브 모듈 → dev client 전용. Expo Go로 실행 불가, Expo Go QR도 없음.** QR은 dev client용.
- 혼란 방지로 `web` 스크립트 제거(wgpu라 웹 미동작).

## 참고 | 알려진 이슈
- `react-native-webgpu` surface `configure`(Metal SwapChain 초기화)가 JS 스레드에서 `MetalView` CALayer를 건드려
  "Modifying properties of a view's layer off the main thread" 경고 발생. 현재 비치명적(렌더됨)이나 실기기/리사이즈 시 불안정 가능 — 실험적 스택의 rough edge.
- 실기기 설치는 Xcode 코드 서명 설정 선행 필요(`No code signing certificates`). 무료 Apple ID 가능(7일 cert).
